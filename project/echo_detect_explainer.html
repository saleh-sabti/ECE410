Section 2

Signal generation — making fake audio

Since we don't have a real microphone here, we fake two audio signals.

Reference signal — what the speaker is playing. We build it as a mix of three sine waves (200 Hz + 440 Hz + 800 Hz) to approximate a voice. Think of it as a simple synthetic vowel sound.

Mic signal — what the microphone picks up. It's the same as the reference, but with a quieter, delayed copy of it added on top (that's the echo) and a little random noise. The formula per sample is: mic[i] = reference[i] + (ECHO_GAIN × reference[i − ECHO_DELAY]) + noise

The hardware is meant to detect that delayed copy. 

def generate_mic(reference):
    mic = []
    for i in range(len(reference)):
        direct = reference[i]
        echo   = ECHO_GAIN * reference[i - ECHO_DELAY]  # delayed copy
        noise  = random.uniform(-NOISE_LEVEL, NOISE_LEVEL)
        mic.append(direct + echo + noise)
    return mic
Section 3

Core algorithm — the actual math

This is the function that does the real work. It takes two lists of N numbers (a window from the reference and a window from the mic) and asks: how similar are these?

It runs three accumulators simultaneously:
  1. dot — multiply each ref sample by the matching mic sample, add them all up. This is the cross-correlation.
  2. energy_ref — square each ref sample, add them up. This measures how loud the reference is.
  3. energy_mic — same thing for the mic.

Divide dot by √(energy_ref × energy_mic) to normalize. Result is between −1 and +1. Close to 1 means the two windows look alike → echo is probably there.

Each accumulator does N multiply-adds = N MACs. Three accumulators = 3×N MACs per window. In hardware, all three run in parallel in a single clock cycle. 

for r, m in zip(ref_win, mic_win):
    dot        += r * m   # N MACs
    energy_ref += r * r   # N MACs
    energy_mic += m * m   # N MACs

# total: 3×N MACs per window call
denom = math.sqrt(energy_ref * energy_mic)
return dot / denom        # score between -1 and +1

Section 4

Streaming loop — processing sample by sample

This is the main loop. It slides a window across the full audio, one sample at a time. At each step: grab the last N samples from both signals, run the correlation, make a decision.

This is exactly how the hardware chip would operate — new sample comes in, oldest sample drops off, output a 1-bit decision. The shift registers in the hardware are doing that "drop oldest, add newest" step. 

for i in range(N, len(reference)):
    ref_win  = reference[i - N : i]   # last N samples
    mic_win  = mic[i - N : i]

    corr     = normalized_xcorr(ref_win, mic_win)
    decision = 1 if corr >= THRESHOLD else 0   # 1-bit output

Section 5

Roofline inputs — is this memory-bound or compute-bound?

For M1 you need to place your algorithm on a roofline chart. That needs two numbers: total FLOPs and total memory traffic.

FLOPs = total MACs × 2 (each MAC = 1 multiply + 1 add = 2 FLOPs).
Memory traffic = for each window, we load 2×N floats (ref + mic) and write 1 float (the score). Each float = 4 bytes. We assume no reuse — every window fetches fresh from memory. That's the worst case.

Arithmetic intensity = FLOPs ÷ bytes. Low number = memory-bound. High number = compute-bound.
